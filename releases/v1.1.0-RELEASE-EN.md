# 🌱 SYSGrow v1.1.0 - Growth Unit Management Gets a Major Upgrade!

**Release Date:** November 9, 2025

## 🎉 What's Cooking?

This release seriously leveled up the Growth Unit Management system, adding physical dimensions, flexible device scheduling, camera integration, and a completely redesigned interface that's easier on the eyes.

---

## ✨ The Big Stuff

### 1. 📏 Physical Dimensions Support (Measuring Up Like a Pro)
Growth units now is tracking pgysical dimensions (width × height × depth in centimeters):
- Define the exact size of your grow space
- Clean dimension display on unit card
- Helps you plan without guessing
- Stored as structured JSON data

**Example:**
```json
{
  "dimensions": {
    "width": 300,
    "height": 250,
    "depth": 200
  }
}
```

### 2. ⏰  Device Scheduling Got Smart
Ditched the rigid light-only schedules for a proper scheduling system:

- **Support for multiple device types**: lights, fans, water pumps, heaters, coolers, humidifiers, dehumidifiers
- **Time-based automation (Set it and forget it)**: Set start and end times for any device (24-hour format)
- **Individual enable/disable (Mix and match)**: Control each schedule independently
- **Night owls welcome**: Schedule devices to run overnight (e.g., 22:00-06:00)
- **Live device monitoring**: Real-time view of currently active devices (See what's running right now)

**New API Endpoints:**
- `GET /api/v1/growth/units/<id>/schedules` - Get all schedules
- `GET /api/v1/growth/units/<id>/schedules/<device_type>` - Get specific schedule
- `POST /api/v1/growth/units/<id>/schedules` - Create/update schedule
- `DELETE /api/v1/growth/units/<id>/schedules/<device_type>` - Remove schedule
- `GET /api/v1/growth/units/<id>/schedules/active` - Get currently active devices

### 3. 📷 Camera Integration (Say Cheese!)
- Enable/disable cameras per growth unit
- Camera status tracking and control
- Time-lapse and monitoring features
- Integration with camera management endpoints

### 4. 🎨 Interface Glow-Up
Complete makeover of the Growth Units page with:
- **Create unit modal** with sections for:
  - Basic info (name, location)
  - Dimensions input (width, height, depth)
  - Device schedule builder with dynamic rows
  - Camera settings
- **Schedule Management Tab** in unit management modal featuring:
  - Visual schedule table with device types and times
  - Add/edit/delete schedule controls
  - Active devices indicator with pulse animation
  - Color-coded device badges
- **Upgraded unit cards** displaying:
  - Physical dimensions with ruler icon
  - Quick stats (plants, sensors, actuators)
  - Environmental data (temperature, humidity)
  - Quick action buttons (Manage, Camera, Schedule)
- **Professional styling**:
  - Responsive data tables with gradient headers
  - Status badges with animations
  - Mobile-friendly design
  - Dark mode support

## 🔧 API Changes

### Updated Endpoints

#### `POST /api/v1/growth/units` - Create Unit
Now accepts additional fields:

**Request Body:**
```json
{
  "name": "Greenhouse A",
  "location": "Greenhouse",
  "dimensions": {
    "width": 300,
    "height": 250,
    "depth": 200
  },
  "device_schedules": {
    "light": {
      "start_time": "06:00",
      "end_time": "22:00",
      "enabled": true
    },
    "fan": {
      "start_time": "08:00",
      "end_time": "20:00",
      "enabled": true
    }
  },
  "camera_enabled": true,
  "custom_image": "/path/to/image.jpg"
}
```

#### `GET /api/v1/growth/units/<id>/thresholds`
Now returns additional fields:
- `device_schedules` - All configured device schedules
- `dimensions` - Physical dimensions of the unit
- `camera_enabled` - Camera status

### New Device Schedule Endpoints

#### Get All Schedules
```bash
GET /api/v1/growth/units/1/schedules
```

**Response:**
```json
{
  "ok": true,
  "data": {
    "device_schedules": {
      "light": {
        "device_type": "light",
        "start_time": "08:00",
        "end_time": "20:00",
        "enabled": true
      },
      "fan": {
        "device_type": "fan",
        "start_time": "09:00",
        "end_time": "21:00",
        "enabled": true
      }
    }
  }
}
```

#### Set Device Schedule
```bash
POST /api/v1/growth/units/1/schedules
Content-Type: application/json

{
  "device_type": "heater",
  "start_time": "22:00",
  "end_time": "06:00",
  "enabled": true
}
```

#### See What's Working Now
```bash
GET /api/v1/growth/units/1/schedules/active
```

**Response:**
```json
{
  "ok": true,
  "data": {
    "current_time": "14:30",
    "active_devices": ["light", "fan"],
    "count": 2
  }
}
```

## 🚀 Getting Started

### For Users

1. **Create a Growth Unit with Full Configuration:**
   - Click "New Unit" button
   - Fill in basic information (name, location)
   - Add dimensions (optional): width, height, depth in cm
   - Configure device schedules (optional): click "Add Device Schedule" to add multiple schedules
   - Enable camera if needed
   - Click "Create Unit"

2. **Manage Device Schedules:**
   - Open any unit's management modal
   - Click the "Schedule" tab
   - View all configured schedules in the table
   - Click "Add Schedule" to create new schedules
   - Edit or delete existing schedules
   - Monitor currently active devices in real-time

### For Developers

**Creating a unit via API:**
```bash
curl -X POST http://localhost:5000/api/v1/growth/units \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Greenhouse",
    "location": "Greenhouse",
    "dimensions": {"width": 300, "height": 250, "depth": 200},
    "device_schedules": {
      "light": {"start_time": "06:00", "end_time": "22:00", "enabled": true}
    },
    "camera_enabled": true
  }'
```

**Managing schedules via API:**
```bash
# Get all schedules
curl http://localhost:5000/api/v1/growth/units/1/schedules

# Set a schedule
curl -X POST http://localhost:5000/api/v1/growth/units/1/schedules \
  -H "Content-Type: application/json" \
  -d '{"device_type": "fan", "start_time": "08:00", "end_time": "20:00", "enabled": true}'

# Get active devices
curl http://localhost:5000/api/v1/growth/units/1/schedules/active
```
