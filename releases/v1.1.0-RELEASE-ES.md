# 🌱 SYSGrow v1.1.0 - La Gestión de Unidades de Crecimiento se Pone las Pilas

**Fecha de Lanzamiento:** 9 de noviembre de 2025

## 🎉 ¿Qué hay de nuevo?

Esta versión le metió ficha al sistema de Gestión de Unidades de Crecimiento, agregándole medidas físicas, horarios flexibles para los dispositivos, cámaras y una interfaz que quedó hecha un papito.

---

## ✨ Lo Más Grosito

### 1. 📏 ¡Ahora con Regla!
Las unidades de crecimiento ahora saben cuánto miden (ancho × alto × profundidad en centímetros):
- Definí el tamaño exacto de tu espacio de cultivo
- Se ve lindo en las tarjetitas de las unidades
- Te ayuda a planificar sin meter la pata
- Se guarda todo en JSON, como la gente

**Ejemplo:**
```json
{
  "dimensions": {
    "width": 300,
    "height": 250,
    "depth": 200
  }
}
```

### 2. ⏰ Los Dispositivos se Ponen la Alarma
Sacamos los horarios fijos de solo luces y metimos un sistema posta:
- **Todos los chiches**: luces, ventiladores, bombas de agua, calefactores, coolers, humidificadores, deshumidificadores
- **Automación a full**: Poné hora de inicio y fin para cualquier dispositivo (formato 24 horas)
- **Prendé y apagá lo que quieras**: Cada horario se maneja por su cuenta
- **Los que trasnochan**: Dispositivos que trabajan de noche (ej: 22:00-06:00)
- **Control en tiempo real**: Mirá qué está laburando en este momento

**Endpoints Nuevos:**
- `GET /api/v1/growth/units/<id>/schedules` - Agarrá todos los horarios
- `GET /api/v1/growth/units/<id>/schedules/<device_type>` - Buscá uno en particular
- `POST /api/v1/growth/units/<id>/schedules` - Creá o actualizá
- `DELETE /api/v1/growth/units/<id>/schedules/<device_type>` - Sacá el que no sirve
- `GET /api/v1/growth/units/<id>/schedules/active` - Fijate quién está laburando ahora

### 3. 📷 ¡Sacá Fotos!
- Activá o desactivá cámaras por unidad
- Controlá el estado de cada cámara
- Soporta time-lapse y monitoreo
- Se integra con los endpoints de gestión de cámaras

### 4. 🎨 La Interface Quedó Re-linda
Le dimos una lavada de cara a la página de Unidades de Crecimiento:
- **Modal para crear unidades** con secciones para:
  - Data básica (nombre, ubicación)
  - Medidas (ancho, alto, profundidad)
  - Armado de horarios con filas que aparecen y desaparecen
  - Config de cámaras
- **Pestaña de Gestión de Horarios** en el modal de gestión con:
  - Tabla visual con tipos de dispositivos y horarios
  - Controles para agregar/editar/borrar horarios
  - Indicador de dispositivos activos que titila
  - Etiquetas de dispositivos con colores
- **Tarjetas de unidades mejoradas** que muestran:
  - Medidas físicas con icono de regla
  - Estadísticas rápidas (plantas, sensores, actuadores)
  - Data ambiental (temperatura, humedad)
  - Botones de acción rápida (Gestionar, Cámara, Horarios)
- **Estilo profesional**:
  - Tablas responsive con headers con degradé
  - Etiquetas de estado con animaciones
  - Diseño que se porta bien en el celular
  - Soporte para modo oscuro

## 🔧 Cambios en la API

### Endpoints Actualizados

#### `POST /api/v1/growth/units` - Crear Unidad
Ahora acepta campos nuevos:

**Cuerpo del Request:**
```json
{
  "name": "Invernadero A",
  "location": "Invernadero",
  "dimensions": {
    "width": 300,
    "height": 250,
    "depth": 200
  },
  "device_schedules": {
    "light": {
      "start_time": "06:00",
      "end_time": "22:00",
      "enabled": true
    },
    "fan": {
      "start_time": "08:00",
      "end_time": "20:00",
      "enabled": true
    }
  },
  "camera_enabled": true,
  "custom_image": "/path/to/image.jpg"
}
```

### Endpoints Nuevos para Horarios

#### Agarrá Todos los Horarios
```bash
GET /api/v1/growth/units/1/schedules
```

**Respuesta:**
```json
{
  "ok": true,
  "data": {
    "device_schedules": {
      "light": {
        "device_type": "light",
        "start_time": "08:00",
        "end_time": "20:00",
        "enabled": true
      },
      "fan": {
        "device_type": "fan",
        "start_time": "09:00",
        "end_time": "21:00",
        "enabled": true
      }
    }
  }
}
```

#### Poné un Horario
```bash
POST /api/v1/growth/units/1/schedules
Content-Type: application/json

{
  "device_type": "heater",
  "start_time": "22:00",
  "end_time": "06:00",
  "enabled": true
}
```

#### Fijate Qué Está Laburando Ahora
```bash
GET /api/v1/growth/units/1/schedules/active
```

**Respuesta:**
```json
{
  "ok": true,
  "data": {
    "current_time": "14:30",
    "active_devices": ["light", "fan"],
    "count": 2
  }
}
```

## 🚀 Para Empezar

### Para los Usuarios

1. **Creá una Unidad de Crecimiento con Toda la Onda:**
   - Clickeá el botón "Unidad Nueva"
   - Completá la data básica (nombre, ubicación)
   - Agregá las medidas (opcional): ancho, alto, profundidad en cm
   - Configurá los horarios de dispositivos (opcional): clickeá "Agregar Horario" para meter varios
   - Activá la cámara si hace falta
   - Clickeá "Crear Unidad"

2. **Manejá los Horarios de los Dispositivos:**
   - Abrí el modal de gestión de cualquier unidad
   - Clickeá la pestaña "Horarios"
   - Mirá todos los horarios configurados en la tabla
   - Clickeá "Agregar Horario" para crear horarios nuevos
   - Editá o borrá horarios existentes
   - Monitoreá los dispositivos activos en tiempo real

### Para los Desarrolladores

**Creando una unidad por API:**
```bash
curl -X POST http://localhost:5000/api/v1/growth/units \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mi Invernadero",
    "location": "Invernadero",
    "dimensions": {"width": 300, "height": 250, "depth": 200},
    "device_schedules": {
      "light": {"start_time": "06:00", "end_time": "22:00", "enabled": true}
    },
    "camera_enabled": true
  }'
```

**Manejando horarios por API:**
```bash
# Agarrá todos los horarios
curl http://localhost:5000/api/v1/growth/units/1/schedules

# Poné un horario
curl -X POST http://localhost:5000/api/v1/growth/units/1/schedules \
  -H "Content-Type: application/json" \
  -d '{"device_type": "fan", "start_time": "08:00", "end_time": "20:00", "enabled": true}'

# Fijate qué está laburando
curl http://localhost:5000/api/v1/growth/units/1/schedules/active
```
